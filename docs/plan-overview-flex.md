# ProjectBrain AI - Gesamtplan (flexibel)

Dieser Plan ist absichtlich modular.
Wir arbeiten in kleinen Steps, koennen aber jederzeit dazwischen:

- brainstormen
- Prioritaeten aendern
- Scope reduzieren oder erweitern
- einen Step splitten

## Arbeitsrhythmus pro Step

1. UI zuerst (sichtbar, klickbar, ohne tiefe Logik)
2. Danach Logik in kleinen Inkrementen
3. Kurz testen
4. Verstehen + reflektieren
5. Erst dann naechster Step

## Step-Dateien

- `plan-step-01-foundation.md`
- `plan-step-02-company-project-ui.md`
- `plan-step-03-project-knowledge-ui.md`
- `plan-step-04-knowledge-logic.md`
- `plan-step-05-kanban-ui.md`
- `plan-step-06-kanban-logic.md`
- `plan-step-07-roles-company-scoping.md`
- `plan-step-08-polish-ai-inbox-prep.md`
- `step-status.md` (aktueller Live-Stand)
- `save-rhythm.md` (wann wir verbindlich aktualisieren)

## Flex-Regeln (wichtig)

- Jeder Step ist ein Vorschlag, kein Zwang.
- Wenn etwas zu gross wird: Step halbieren.
- Wenn Lernen stockt: Tempo reduzieren, mehr Erklaerung, weniger Umfang.
- Wenn Motivation auf ein anderes Feature geht: Step-Reihenfolge anpassen.
- "Done" bedeutet: Du verstehst den Flow, nicht nur dass Code laeuft.
- Nach jedem abgeschlossenen Mini-Step wird `step-status.md` aktualisiert.
